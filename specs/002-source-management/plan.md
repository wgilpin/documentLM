### Implementation notes

#### 1. `PdfParseError` in service layer

Add a new exception class and raise it (instead of swallowing) when `pypdf.PdfReader` fails:

```python
class PdfParseError(Exception):
    pass

# In _extract_pdf_text тАФ raise PdfParseError on exception
# In add_source_pdf тАФ let PdfParseError propagate up
```

#### 2. API error handling

In `sources.py`, the `add_source` endpoint catches `PdfParseError` and returns a 422 HTML fragment (for HTMX requests) or raises `HTTPException(422)` for JSON requests.

#### 3. Empty state in list endpoint

In `list_sources` endpoint: when HTMX request and result is empty, return:

```html
<li class="source-empty-state">No sources added yet.</li>
```

#### 4. Template changes

- `document.html`: rename heading, add `id="source-error-{type}"` containers adjacent to each form
- `partials/sources.html`: add `source.created_at.strftime('%b %d, %Y')` and restructure layout per contract
