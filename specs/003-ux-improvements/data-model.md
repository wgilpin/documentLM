# Data Model: UX Improvements

**Branch**: `003-ux-improvements` | **Date**: 2026-03-13

## Schema Changes

### New Table: `chat_messages`

Stores conversational turns for the meta-chat panel. Scoped to a document; history is visible within the session that created it (no cross-session UI, but rows persist in DB).

| Column        | Type                | Constraints                              | Notes                              |
| ------------- | ------------------- | ---------------------------------------- | ---------------------------------- |
| `id`          | UUID (PK)           | NOT NULL, default `uuid4()`              | Surrogate primary key              |
| `document_id` | UUID (FK)           | NOT NULL, CASCADE DELETE → `documents`   | Scopes chat to a document          |
| `role`        | ENUM `ChatRole`     | NOT NULL                                 | `user` or `assistant`              |
| `content`     | TEXT                | NOT NULL                                 | Message body                       |
| `created_at`  | TIMESTAMPTZ         | NOT NULL, server default `now()`         | For ordering turns                 |

**Enum: `ChatRole`**
- `user` — message sent by the human author
- `assistant` — response generated by `ChatAgent`

### No Changes to Existing Tables

The following existing entities already have everything needed:

- **`suggestions`**: Position data comes from the parent `comments` row via `comment_id` FK (which stores `selection_start` / `selection_end`). No new columns required.
- **`comments`**: Already stores `selection_start`, `selection_end`, `selected_text`, `body`. The contextual floating menu and command modal both submit to the existing `POST /api/documents/{doc_id}/comments` endpoint.
- **`sources`**: No schema change; the collapse and hover-delete changes are purely frontend.
- **`documents`**: No change.

## ORM Model (SQLAlchemy)

```python
class ChatMessage(Base):
    __tablename__ = "chat_messages"

    id: Mapped[uuid.UUID] = mapped_column(UUID(as_uuid=True), primary_key=True, default=uuid.uuid4)
    document_id: Mapped[uuid.UUID] = mapped_column(
        UUID(as_uuid=True), ForeignKey("documents.id", ondelete="CASCADE"), nullable=False
    )
    role: Mapped[ChatRole] = mapped_column(Enum(ChatRole), nullable=False)
    content: Mapped[str] = mapped_column(Text, nullable=False)
    created_at: Mapped[datetime] = mapped_column(
        DateTime(timezone=True), server_default=func.now(), nullable=False
    )
```

### No Schema Changes Required

`SuggestionResponse` does not need `selection_start`/`selection_end` because the inline overlay does not use character-level pixel positioning (see [research.md](research.md)).
